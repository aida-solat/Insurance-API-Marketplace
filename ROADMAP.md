# Roadmap

## Near term

- Harden repository security posture and keep dependencies patched.
- Expand test coverage for decision flows and provider integrations.
- Publish deployment runbooks for self-hosted and cloud setups.
- Improve contributor onboarding and sponsor-facing project documentation.

## Product direction

- Add more insurance product templates and richer underwriting rules.
- Expand claims triage with stronger fraud signals and evidence workflows.
- Make model comparison and audit trail inspection easier in the UI.
- Add operational dashboards for latency, provider health, and decision volume.

## Community

- Grow the issue backlog into clearly labeled starter, help-wanted, and roadmap
  items.
- Document release and support expectations as the project matures.
