# Security Policy

## Supported Versions

Security fixes are applied on the active default branch and shipped in the next
release or deployment cycle.

| Branch | Status |
| ------ | ------ |
| `main` | Supported |

## Reporting a Vulnerability

Please do **not** open a public issue with exploit details.

1. Use GitHub's **private vulnerability reporting** for this repository when it
   is available in the GitHub UI.
2. If private reporting is not available to you, contact the maintainer through
   the repository owner's GitHub profile and share only the minimum details
   needed to establish a secure channel.

Include:

- affected endpoint, page, or package
- impact and attack prerequisites
- reproduction steps or proof of concept
- suggested remediation if you have one

We aim to acknowledge reports within 3 business days and follow up with status
updates until the issue is resolved.
